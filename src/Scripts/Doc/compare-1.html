1. compare

1.1 NAME

compare

1.2 SYNOPSIS

/usr/local/seq/bin/compare old getlocs file new getlocs file

1.3 DESCRIPTION

This program compares getlocs outputs from two of a projects splice assemblies that have been made at different times. Typically, mat numbers assigned by splice are highly sensitive to the details of the input sequences, making comparison difficult because many otherwise unchanged mats will receive different numbers in the assembly. compare reassigns mat numbers in the new getcmnts file to reflect similarities and differences between the new and old assemblies.

While the getlocs files contain much assembly information, compare simply considers mats sets of sequences and analyzes old and new assemblies by examining set memberships. Sequences are considered the same if and only if their gel-lane designations are the same --- All other name information is disregarded. If a sequence has been determined to be a chimera and has been split into two new sequences with a and b extensions, the sequences are considered different from the original.

compare detects the following relationships between old and new mats:

  • equivalences

    A mat in the old assembly and a mat in the new assembly are considered equivalent if they both have exactly the same set of sequences (barring sequences which have been removed from or added to the project in the time between the two assemblies).

  • fusions

    A mat in the new assembly is a fusion of two or more mats in the old assembly if and only if the sequence set of the new mat is exactly the sum of the sequence sets in the old mats (again, disregarding sequences which have been removed from or added to the project after the time of the old assembly).

  • fissions

    A mat from the old assembly fissions into two or more mats in the new assembly if and only if the sequence set of the old mat is exactly the sum of the sequence sets in the mats (once again, disregarding sequences that have been removed from or added to the project after the old assembly).

  • appearances

    A mat in the new assembly appears if none of its member sequences are present anywhere in the old assembly (i.e. if all of its sequences have been added after the old assembly was made).

  • disappearances

    A mat in the old assembly disappears if none of its member sequences are present anywhere in the new assembly (i.e. if all its sequences have been removed after the old assembly was made).

  • complexes

    These are combinations of partial fusions and fissions. Basically, these are mats which do not fall into any of the above categories. A mat from the old assembly is either completely contained by a single mat in the new assembly, or splits into two or more mats in the new assembly.

1.4 AVAILABILITY

Currently, on genome5, genome4, genome1, ospa, ospb

1.5 CAVEATS

Mat-renaming capabilities have not been added to the new version.

The current version does not recognize mat names like block11a or new1. It therefore will not accept one of it's own output files as input. This can be fixed if necessary.
